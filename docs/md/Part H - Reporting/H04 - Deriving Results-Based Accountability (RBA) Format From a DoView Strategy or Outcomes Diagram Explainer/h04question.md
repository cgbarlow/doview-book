# Is Results-Based Accountability (RBA) compatible with the DoView Planning approach?

> **Pair:** Question (this page) · [Tool](h04tool.md)

Results Based Accountability (RBA) is one of several different indicator frameworks used in the public sector. Results Based Accountability includes two aspects. The first of these is that it is a set of headings under which an indicator set can be organized. The headings are based on three questions.

How much did we do?
How well did we do it?
Is anyone better off?

To use the Results Based Accountability approach with a DoView strategy/outcomes diagram you just need to take the indicators identified from a the DoView diagram and group them under the Results Based Accountability headings.

In this Deriving a Results Based Accountability Format From a Visual Outcomes Diagram Explainer you can see that the indicators from the Archery Initiative (B4) DoView have been reorganized under the three Results Based Accountability questions.

In suitable software*, you can save time by simply 'cloning' these indicators from where they appear on the DoView diagram as clones and putting them onto a separate page grouped under the three Results Based Accountability questions. Then if you update them in one place they will be updated in both.

The second aspect of Results Based Accountability is called Turning The Curve. This is a process where stakeholders discuss what needs to be done in order to improve shared high-level outcomes.

DoView Planning provides a systematic and robust method for holding such discussions by doing it against a DoView strategy/outcomes diagram setting out the outcomes and steps it is believed will lead to them for the relevant sector.

\* For instance the DoView Planning legacy app.
