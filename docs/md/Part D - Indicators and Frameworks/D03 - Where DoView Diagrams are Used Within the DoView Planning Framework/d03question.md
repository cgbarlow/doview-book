# How can the building blocks of any outcomes system be undertaken by using DoView diagrams?

> **Pair:** Question (this page) · [Tool](d03tool.md)

The DoView Planning Framework (D1) comprehensively outlines the building block components needed within any well-formed outcomes system. In outcomes theory, an outcomes system is any system designed to identify, prioritize, align, implement, measure, or hold parties to account for achieving outcomes of any sort in any sector. DoView Planning provides tools to help you work within any of the Framework's building blocks. DoView Planning tools are fully integrated, so work done with one tool within any Framework component can be leveraged in others.

Traditionally, when working on a particular component in the Framework for a particular organization or initiative, you could find yourself repeating work already done in another Framework component. However, if you use DoView Planning's integrated toolset, it avoids you repeating any work. This is particularly so in regard to specifying outcomes and the steps leading to them which in DoView Planning is done once in the form of a DoView diagram used to underpin all of the steps in the DoView Planning Framework.

There are three Strategy and Planning components in the Framework. Once you've built a DoView strategy/outcomes diagram, you can use it for identifying your 'This-Then' logic, mark up priorities, and check for alignment between projects/activities and steps/outcomes.

The two Monitoring Performance and Outcomes components are best done against the relevant DoView diagram. It shows exactly where indicators, controllable or not, sit within the relevant DoView diagram.

The four Evaluation components all benefit from having the underlying DoView diagram. In Implementation evaluation, you can compare what is supposed to happen (lower levels of the DoView) with what actually is happening. Process evaluation gives more detail of what's occurring in the initiative. Impact evaluation benefits from visualizing the level at which you are assessing impact. Economic and summative evaluation benefit from clearly defining the outcomes included in your analysis. The Where DoView Diagrams are Used Within the DoView Planning Framework Explainer (D3) sets this out.
