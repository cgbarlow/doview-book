# How can we conceptualize the relationship between strategy, alignment, deliverables, indicators, performance improvement, evaluation and value for money within an 'outcomes system'?

> **Pair:** Question (this page) · [Tool](d1tool.md)

A clear, integrated conceptual framework is needed to fully understand the links between organizational tasks such as strategy, alignment, outputs (deliverables) specification, indicator measurement, performance improvement, evaluation, and value for money. The DoView Planning Framework provides a way of understanding the relationship between these tasks when undertaken by any agency, provider, policy, organization, or initiative.

Within the DoView Planning Framework there are three Strategy & Planning components, Component A captures the initiative's underlying rationale and logic (drawing on any prior evidence); Component B selects, for the current planning period, the priority steps and outcomes from A that will be emphasized; and Component C demonstrates "line-of-sight" alignment between those priorities and the initiative's outputs, deliverables, activities, or projects. These are followed by two Monitoring, Performance and Outcomes components, Component D tracks outputs and other controllable measures that can be directly attributed to the initiative, while Component E monitors higher-level outcomes and other not-necessarily controllable measures that cannot simply be attributed to the initiative. Finally, there are four Evaluation components in the framework, with Component F focusing on implementation evaluation to optimize delivery, Component G on process evaluation to document and analyze course and context, Component H on impact evaluation to attribute changes in high-level outcomes to the initiative, and Component I on economic and summative evaluation to demonstrate value for money and the overall worth of the initiative.

The DoView Planning Framework is also known as the Outcomes System Components (or Building Blocks) Diagram. Any accounting system includes a set of building blocks (e.g. a general ledger and a depreciation schedule). In the same way, the DoView Planning Framework sets out the building blocks that are required in any well formed outcomes system.
