# DoView Tool I1 — Steps in Introducing DoView Strategy/Outcomes Diagrams and DoView Planning Into an Organization, Initiative or Sector

> **Pair:** [Question](i1question.md) · Tool (this page)

Below are the steps you need to take when introducing DoView strategy/outcomes diagrams and DoView Planning. These show how DoView diagrams can progressively be adopted as a central tool in planning and implementation. The idea is for an organization or initiative's DoView to become the shared thinking tool (artefact) for all planning, implementation and measurement discussions. Having a shared thinking tool for planning is analogous to military planners using a map when discussing strategy, or when those discussing a building project talk about their ideas against a detailed plan of the building.

Steps in progressive implementation of DoView strategy/outcomes diagrams and DoView Planning.

## Diagram

```mermaid
flowchart TD
    s1["Introducing the idea of using DoView strategy/outcomes diagrams built using the DoView Drawing Rules (B7)"]
    s2["Working bottom-up — showing how DoView Planning can make planning, implementation and measurement much easier, faster, more integrated and aligned"]
    s3["Working top-down — in both the public and private sectors showing politicians, CEOs, management, purchasers/funders that DoView Planning makes oversight easier"]
    s4["Introducing DoView diagrams subpages into existing strategy and reporting documentation setting the scene at the start of each chapter in such reports"]
    s5["Ensuring that the ICT infrastructure is available for DoView Planning, initially in PowerPoint/Google Slides/DoView Planning legacy app then using new app/platforms/bespoke systems incorporating DoView Planning's feature set"]
    s6["Getting people used to the discipline of always working with a DoView diagram when discussing any aspect of strategy, planning, implementation and measurement"]
    s7["Making any procedural, regulatory, or other changes required to further facilitate using DoView diagrams for all aspects of planning, implementation and reporting"]
    s8["Ensuring that DoView diagrams form the basis of formal planning, implementation and reporting at all levels"]
    s1 --> s2 --> s3 --> s4 --> s5 --> s6 --> s7 --> s8
```
