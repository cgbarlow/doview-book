# How can an organization or initiative deal with meeting multiple outcomes demands from different control agencies, purchasers/funders or other stakeholders?

> **Pair:** Question (this page) · [Tool](b18tool.md)

Often an agency, provider, policy, organization or initiative has to deal with, or report on, multiple outcomes sets from control agencies, purchasers, funders, or other stakeholders. Sometimes they also need to meet sector-wide outcomes. These various requirements are described as: outcomes, standards, requirements, best practice, quality certification, targets, goals, elements in letters of expectation, etc. Dealing with multiple overlapping outcomes sets is demanding.

In DoView Planning, all of these different 'outcome sets' are treated in the same way. Any outcomes sets an initiative needs to meet are visually modeled as a set of boxes in a DoView strategy/outcomes diagram and the initiative's activities or projects visually mapped onto relevant boxes in the multiple outcomes sets using DoView Visual Alignment. When using an app or platform with the right features the analysis phase of this work can be done simply by dragging links between the activity or project boxes and the 'outcome boxes', they are attempting to influence in the different outcomes sets. This shows the visual line-of-sight alignment with the multiple outcomes sets. AI can be used to check on which boxes are likely to influence which outcomes. It is then a simple task to get AI to put these results into whatever format a control agency, purchaser or funder requires.

Ideally, over time (particularly if they are using DoView Planning) control agencies, purchasers, funders or other stakeholders will get used to working directly with strategy/outcomes diagrams and realize that the fastest and most robust way to identify if an initiative is focused on a particular 'outcomes set' is not to ask for a text and table-based report. Instead, it is to visually review the relevant interactive DoView strategy/outcomes diagram for line-of-sight alignment between the initiative's activities and projects and the boxes in the particular outcomes set. Using this approach also means that an initiative can continue to structure its own outcomes set independently of external demands. This How An Initiative Can Deal With Multiple Outcomes Sets Tool (B18) shows how an initiative can map its activities or projects onto its own outcomes set, in addition to multiple others. Note that this only shows the fact that the initiative is 'focused' on a particular outcomes set, in contrast to proving at this stage that the initiative has actually changed them.
