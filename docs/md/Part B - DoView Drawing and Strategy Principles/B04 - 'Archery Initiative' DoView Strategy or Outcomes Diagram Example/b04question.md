# Is there a simple example of a DoView strategy/outcomes diagram that can be used?

> **Pair:** Question (this page) · [Tool](b04tool.md)

This 'Archery Initiative' DoView Drill-Down Strategy/Outcomes Diagram Example (B4) provides a simple example of a DoView strategy/outcomes diagram drawn according to the DoView Drawing Rules Tool (B7). You can use this example to show people what a simple mock-up DoView strategy/outcomes diagram looks like.

This mock-up DoView is also used in later tools within this book to show the ways in which DoView strategy diagrams can be used. Of course, DoView strategy/outcomes diagrams for real-world initiatives are usually much more complex than this example. To deal with this complexity, you usually build real-world DoView strategy/outcomes diagrams as layered drill-down diagrams. This means they communicate the complexity of an initiative, but do so in a way that still lets a reader quickly overview and understand the diagram.

In real world examples, an overview page is used to provide a 'helicopter' view of the DoView, and the reader can then drill-down into any of the lower-level pages to get as much detail as they wish. You will find examples of real world DoViews at DoViewPlanning.Org. Please look at some of these now to get a feel for how real-world DoView strategy/outcomes diagrams work. Note how they include drill-down pages, whereas the Archery Example is just a single page to keep it simple.

An important feature of the DoView strategy/outcomes diagrams used in DoView Planning is that ideally they should be used interactively. Printed versions can be included in documents and reports. However, to get the most value from DoView strategy/outcomes diagrams, they need to able to be drilled-down in real-time during meetings. This can be done if they are presented in apps like PowerPoint or Google Slides or in the DoView Planning legacy app. Being able to drill-down into a DoView means that users, whether at their own computer or in a meeting, can interact with, and immediately edit, the DoView. Users should also be able to interactively interrogate the DoView to identify different aspects of planning. For instance, by using DoView Visual Alignment to check for line-of-sight alignment between activities/projects and higher-level priority outcomes.
