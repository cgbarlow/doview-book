# Why don't DoView strategy/outcomes diagrams show all causal links?

> **Pair:** Question (this page) · [Tool](b08tool.md)

When used within DoView Planning, DoView strategy/outcomes diagrams are represented using a simple set of visual conventions. The initial set of conventions has been deliberately kept simple in the early stages of encouraging the widespread adoption of DoView Planning. This is so that DoViews are accessible by a wide range of potential users. As people use DoViews more it will be possible to introduce a richer set of conventions. DoView diagrams should be seen providing a visual language for articulating strategy which will become more sophisticated over time. New developments such as 3-D representations in virtual environments will assist this. At the moment a DoView used by humans can be based on an underlying DoView in which AI represents and works with more information, for instance all of the links between different boxes. Key current conventions are discussed below.

Fuzzy causality - Just show general left to right causality and a single causual arrow between columns of boxes within a diagram. While one can attempt to spell out the full set of relationships between boxes within a DoView, a fully articulated 'wiring diagram' is highly complex and off-putting for most human readers. Given the formatting requirements of diagrams with a large number of links they also do not invite people to edit them because doing so 'breaks' the tidy visual formatting of causal lines. In DoView Planning, DoView diagrams are seen as 'living documents' that invite amendment so that they will end up capturing the consensus mental model of 'This-Then' strategy lying behind any planning and implementation effort.

No feedback loops shown - Feedback loops are not normally shown in DoViews in order to keep the representation simple. If thought necessary, a note saying 'Not all links and feedback loops are shown' can be included at the bottom of any page.

'This-Then' versus 'And' statements - A filled arrow is used to show a 'This-Then' relationship between at least one box in a column with at least one box in the next column. An arrow which is not filled is used to indicate just an 'And' statement. In effect this makes the second column simply a continuation of the first.
