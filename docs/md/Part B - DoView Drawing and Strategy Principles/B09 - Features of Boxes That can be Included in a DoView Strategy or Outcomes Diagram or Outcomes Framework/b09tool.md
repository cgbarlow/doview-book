# DoView Tool B9 — Features of Boxes That can be Included in a DoView Strategy/Outcomes Diagram or Outcomes Framework

> **Pair:** [Question](b09question.md) · Tool (this page)

Use this list of features to clarify any discussion about what should, or should not, be included within a DoView strategy/outcomes diagram or outcomes framework. 'A' lists the possible features of the types of boxes that can be included in DoView diagrams or outcomes frameworks. 'B' shows how these can be used to provide a tight technical definition of a particular term such as: 'goal', 'outcome', 'result' etc.

## A — Possible features

| Feature | Definition |
|---|---|
| RELEVANT | Relevant to achieving higher-level outcomes within the DoView or outcomes framework for the initiative. It may be something the initiative controls, can influence, or a risk or assumption even if outside the initiative's control. |
| INFLUENCABLE | Can be influenced by what the initiative does. |
| CONTROLLABLE | Only influenced by the initiative, not by other factors. |
| MEASURABLE | Able to be measured. |
| ATTRIBUTABLE | Able to be attributed to the particular initiative (i.e. you can prove that the particular initiative made it happen rather than other factors). |
| ACCOUNTABILITY | Something the initiative will be rewarded or punished for. |
| APPROPRIABLE | A party can capture the monetary value of achieving it. |
| LEVEL | An element can be towards the top of a strategy/outcomes diagram or outcomes framework (high-level) or lower down. |

## B — Using the features to define terms

Using this list, a control purchaser/funder or supplier/provider can specify exactly what they mean by any term they are using, for example:

- 'An "outcome" as we are using the term is: RELEVANT, INFLUENCABLE, NOT-NECESSARILY CONTROLLABLE, MEASURABLE and HIGH-LEVEL.'
- 'A "deliverable" as we are using the term is: RELEVANT, INFLUENCABLE, CONTROLLABLE, MEASURABLE, ATTRIBUTABLE and ACCOUNTABILITY.'
