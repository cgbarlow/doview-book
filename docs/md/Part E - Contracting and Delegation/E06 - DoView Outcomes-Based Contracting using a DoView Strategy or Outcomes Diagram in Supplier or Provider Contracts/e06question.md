# What is the best way of ensuring that deliverables in a supplier/provider's contract is aligned with higher-level outcomes?

> **Pair:** Question (this page) · [Tool](e06tool.md)

Provider contracts typically include a deliverables list (outputs, service standards, KPIs list). It is essential that there is alignment between such deliverables lists and the high-level outcomes sought by the purchaser/funder.

You can use DoView Outcomes-Based Contracting to achieve such alignment by attaching the relevant DoView strategy/outcomes diagram to supplier/provider contracts. Specific deliverables can be marked up on the DoView, showing what the supplier/provider is responsible for under the contract. For more discussion on using a DoView diagram to identify deliverables, see Developing or Reviewing Indicators, Deliverables, Outputs or KPIs Lists Tool (D10). Depending on the level at which the DoView diagram the purchaser/funder is using has been struck, it may be necessary to draw additional lower-level drill-down DoView diagram subpages to detail steps that a particular supplier/provider is being contracted to do.

Using the DoView diagram to specify deliverables is also a way of preventing 'slippage' within the purchaser/funder's documentation. Purchasers/funders will have discussions and develop documentation specifying the outcomes they are seeking. If they hold internal or stakeholder discussions against the relevant DoView and then use the same DoView within a contract, no slippage will occur. Slippage occurs where there is a failure to faithfully translate high-level purchaser/funder outcomes documentation into specific deliverables within a contract. Using deliverables marked up onto the relevant DoView strategy/outcomes diagram attached to a contract also means that a supplier/provider is aware of where their deliverables fit within the wider context of the steps and outcomes in the DoView diagram. It is a good idea for any supplier/provider to be aware of this outcomes context and, in addition, the marked-up DoView diagram plays a central role when using Type 3 contracting (for outputs, higher-level controllable indicators plus 'managing for outcomes') from the Types of Contracting and Delegating for Outcomes or Outputs Tool (E3). Note that if suppliers/providers have different contracts with purchasers/funders, they can use the How An Initiative Can Deal With Multiple 'Outcomes Sets' Tool (B18) to manage these efficiently.
