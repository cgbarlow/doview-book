# DoView Tool E1 — Types of Reporting and Accountability a Control Agency or Funder Can Require From an Initiative

> **Pair:** [Question](e01question.md) · Tool (this page)

This tool helps a central agency/purchaser/funder to decide on and communicate exactly what it wants from an agency, provider, policy, organization or initiative in terms of reporting and accountability. The control agency/purchaser/funder can get an initiative itself to provide any of the components below; the central agency or funder can provide one or more of them itself; or it can get other parties to provide some of the components. See DoView Planning Framework (D1) for more detail on these components.

In the example below, the organization that will provide each component from the DoView Planning Framework (D1) is shown with the box in gray. More than one party can do one of these components. For instance, the initiative improvement component is being done by the initiative and another party (e.g. a quality assurance organization or an implementation evaluator).

## Diagram

The example assignment below uses ✓ to mark the gray ("assigned") boxes. Each component (A–I) can be assigned to the initiative, the funder, or another party — and may be shared.

| Component | Initiative will do | Funder will do | Another party will do |
|---|---|---|---|
| A — Strategy | ✓ | | |
| B — Prioritize | ✓ | | |
| C — Align | ✓ | | |
| D — Outputs (controllable) | ✓ | | |
| E — Outcomes (not-necessarily controllable) | ✓ | | |
| F — Implementation optimization & evaluation | ✓ | | ✓ |
| G — Process evaluation (description) | ✓ | | |
| H — Impact evaluation | ✓ | | |
| I — Economic and summative evaluation | | ✓ | |

In this example the initiative does almost everything itself; the funder takes responsibility for component I (economic and summative evaluation); and another party (e.g. a quality assurance body or implementation evaluator) shares component F with the initiative.
