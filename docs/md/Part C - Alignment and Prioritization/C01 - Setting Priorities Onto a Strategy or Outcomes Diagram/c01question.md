# How can agencies, organizations and initiatives set their priorities using DoView strategy/outcomes diagrams?

> **Pair:** Question (this page) · [Tool](c01tool.md)

In any planning period (e.g. typically a year), not everything that it is desirable to do will translate into a priority for an initiative because it will need to take into account cost and feasibility considerations. Using the DoView Planning approach, what both politicians and government agencies want to do will be set out in one or more visual strategy/outcomes diagrams. Also, citizens' outcomes will be included within these or set out in separate strategy/outcomes diagrams. (See the Politicians, Government Agency Business-As-Usual and Citizens' Outcomes Integration Tool (B21)).

Using this approach, priorities are set by politicians and government agencies working through one or more sets of strategy/outcomes diagrams. They simply mark-up the boxes in the diagrams that they regard as current priorities. Priorities are typically set as A, B, C or BAU (Busines-As-Usual for ongoing, necessary work).

There are two considerations that need to be taken into account when setting priorities. The first is what has been achieved on the box so far and whether further work needs to be done on it to achieve the outcomes included in the relevant strategy/outcomes diagram. The second is whether, given the resources you have, the particular box is a relative priority compared to the other things that the government could spend its funds on in the current planning period. Typically, priorities are set in a workshop where decision-maker participants are asked to consider these two factors at the same time and reach a consensus on what the priorities are for the current planning period.

Within the Government Planning and Implementation Cycle (A2), the priority setting process starts by setting an initial set of priorities based on an analysis of the options and trade-offs (see the Government Planning and Implementation Cycle Step 4). Then the deliverables needed to make these priorities happen are worked up together with how much it will cost to deliver them. Following this stage, decision makers go back and reassess the initial priorities in terms of the resources available and the estimates that have been provided for the cost of the deliverables (this is set out in the Government Planning and Implementation Cycle Steps 1 - 7).
