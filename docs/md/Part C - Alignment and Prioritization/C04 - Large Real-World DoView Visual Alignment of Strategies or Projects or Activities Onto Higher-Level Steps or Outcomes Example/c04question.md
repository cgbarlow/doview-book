# How is DoView Visual Alignment scalable to large real-world organizations, providers, policies and initiatives?

> **Pair:** Question (this page) · [Tool](c04tool.md)

The DoView Visual Alignment Tool (C3) shows how strategies, activities or projects can be mapped onto higher-level outcomes in a simple DoView strategy/outcomes diagram. But what about large real-world agencies, providers, policies, organizations or initiatives that may have a hundred or so strategies/activities/projects and many steps/outcomes?

As has been mentioned, if you want to do DoView Visual Alignment efficiently you really need software that includes suitable features\*. In particular providing a count of how many strategy/activity/project boxes are linked to how many higher-level step or outcome boxes and allowing you to interact with any box within the diagram to see a list of the boxes that are linked with it.

In addition, as has been noted already, to be able to practically work with large strategy/outcomes diagrams, you need to be able to break your diagram up into separate drill-down subpages. This is so that it can be easily understood by readers, able to be worked with online, used on a tablet, dataprojected onto a standard-sized screen, and fitted onto a normal-sized printed page.

The app or platform you use needs specialist features. For instance, allowing you to make links between boxes situated on different pages (e.g. step/outcome drill-down pages and a page with strategies/activities/projects listed on it). It also needs a way of showing the same boxes on different pages with the links represented in different ways.

Conceptually, apps or platforms need to move beyond the idea that a link can only be represented in a single way - by a line. This contrasts with traditional drawing apps which normally only represents links as line-links. Multiple ways of representing a link allows you to work with large-scale diagrams. You initially make the linking between boxes on different pages then provide another representation showing a single poster version using line-links\*.

The Real-world Mapping of Strategies/Projects/Activities Onto Higher-Level Steps/Outcomes Example (C4) shows a scalable real-world example.

\* The DoView legacy app has suitable features.
